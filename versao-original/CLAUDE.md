## Project Overview

Bilingual (PT/EN) Bible study website deployed on Cloudflare Pages via the Astro framework. The site covers all 66 canonical Protestant books, with deep-dive sub-pages for Old Testament sections.

- **PT** — Portuguese (NAA translation), default language.
- **EN** — English (ESV translation), native translated pages.

Live site: `https://estudobiblico.pages.dev`

## Commands

All commands run from `astro-site/`:

```bash
npm run dev          # Dev server at http://localhost:4321
npm run build        # Production build to dist/
```

Deploy is automatic: push to `main` → Cloudflare Pages picks it up. No manual deploy step needed.

After adding or editing sub-page HTML source files, regenerate Astro pages:

```bash
node scripts/extract-subpages.mjs
```

## Architecture

### Index pages (homepage grid + modal)

`pt/index.astro` and `en/index.astro` load all 66 books from the content collection at build time, serialize them to JSON via `define:vars`, and render an interactive grid. Clicking a cell calls `window.show(id)` (client-side JS) which populates and opens an inline modal — no page navigation.

**Critical:** Astro wraps `define:vars` scripts in an IIFE. Any function called from an inline `onclick` must be assigned to `window.show`, `window.close_`, etc. — not declared with `function foo()`.

**CSS scoping trap:** Local `<style>` blocks in `.astro` pages get scoped with a `data-astro-cid-*` attribute. If a scoped rule targets `.cell`, it overrides global rules from `BaseLayout.astro` with higher specificity. Never put `border: none` or other resets in a local `<style>` that would override BaseLayout's cell styling.

Each index page also writes its locale to `localStorage` on load so the root redirect at `/` remembers the last-used language.

### Sub-pages (at-pentateuco, at-historicos, etc.)

Sub-pages are generated by `extract-subpages.mjs` from the legacy HTML files in the repo root. The script:

1. Reads `at-{slug}/index.html` for the PT version.
2. Checks if `at-{slug}/index.en.html` exists — if yes, uses it as the native EN source; if no, falls back to the PT HTML (bilingual via `data-pt`/`data-en` attributes).
3. Fixes back-links (`../index.html` → `/pt` or `/en`).
4. Writes `.astro` files that embed the full HTML as a JSON string in `const html = ...` + `<Fragment set:html={html} />`.

Never edit `src/pages/pt/at-*.astro` or `src/pages/en/at-*.astro` directly — they are overwritten by the script. Always edit the source HTML files in the repo root, then re-run the script.

**Language toggle pattern for sub-pages:** Sub-pages use full-page redirect (not DOM attribute swapping) because content is fixed HTML with no `data-pt`/`data-en` attributes.

- PT source (`index.html`): shows "EN" button → `toggleLang()` sets `localStorage('lang','en')` + redirects to `/en/at-{slug}`
- EN source (`index.en.html`): sets `localStorage('lang','en')` on load, shows "PT" button → `goToPt()` sets `localStorage('lang','pt')` + redirects to `/pt/at-{slug}`

```javascript
// In index.html (PT page)
function toggleLang() {
  localStorage.setItem('lang', 'en');
  window.location.href = '/en/at-{slug}';
}
document.getElementById('langLabel').textContent = 'EN';

// In index.en.html (EN page — inline script near </body>)
localStorage.setItem('lang', 'en');
function goToPt() {
  localStorage.setItem('lang', 'pt');
  window.location.href = '/pt/at-{slug}';
}
```

### Content collection (book data)

Each book is a JSON file in `src/content/books/pt/` or `src/content/books/en/`. Schema:

```json
{
  "id": "gn",
  "num": "Book 1 · Old Testament",
  "title": "Genesis",
  "abbr": "Gen · Pentateuch · Law",
  "cellAbbr": "Gen",
  "author": "Moses (Mosaic tradition)",
  "chars": ["Adam", "Eve", ...],
  "tags": ["Creation", "Covenant", ...],
  "body": "<p>...</p>",
  "verse": "\"In the beginning...\" <cite>Genesis 1:1 — ESV</cite>",
  "translation": "ESV",
  "order": 1,
  "category": "pen"
}
```

- `cellAbbr` — abbreviation shown in the grid cell. PT uses Portuguese abbreviations (Gn, Êx, Sl…); EN uses ESV/SBL abbreviations (Gen, Exod, Ps…).
- `num` — shown in modal header. Must be in the file's language ("Livro 1 · Antigo Testamento" for PT, "Book 1 · Old Testament" for EN).
- `category` — one of: `pen his poe mai men eva ato pau ger apo`. Controls cell color.
- `translation` — `"NAA"` for PT files, `"ESV"` for EN files.

### Styling

Global CSS lives in `BaseLayout.astro` (`<style is:global>`). Key variables:

```css
--navy: #2c2416       /* background */
--navy-mid: #3a3020   /* sidebar, toolbar */
--gold: #FFA400       /* primary accent */
--sand: #f0e6d0       /* body text */
--cell-border: rgba(255,164,0,.55)  /* book cell border */
```

Sub-pages load `shared.css` from the repo root (served as a static asset). Per-page accent color is set via a `:root { --accent: ... }` inline style injected by `SubpageLayout.astro`.

### Quotation conventions

- PT: NAA translation, cited as `— NAA`. LORD rendered as `<span style="font-variant:small-caps">Senhor</span>`.
- EN: ESV translation, cited as `— ESV`. Chapter:verse separator is a colon (`Genesis 1:1`), not a period.
